### 2.2 流程一：语音快速生成（最常用路径）

```
首页
  │
  ├─ [用户按下录音按钮]
  │     │
  │     ├─ 检查麦克风权限
  │     │     ├─ 未授权 → 权限引导 → 用户授权/拒绝
  │     │     │                 ├─ 授权 → 继续录音
  │     │     │                 └─ 拒绝 → 提示切换键盘输入
  │     │     └─ 已授权 → 开始录音
  │     │
  │     ├─ [用户说话中...]
  │     │     实时声波动画
  │     │     时长计时器（最长 5 分钟）
  │     │
  │     ├─ [用户松开/点击停止]
  │     │     │
  │     │     ├─ 录音时长 < 1 秒 → 忽略（防误触）
  │     │     └─ 录音时长 ≥ 1 秒 → 进入语音识别
  │     │
  │     ├─ 语音识别中...
  │     │     转文字进行中（loading）
  │     │
  │     ├─ [识别完成]
  │     │     展示转文字结果
  │     │     │
  │     │     ├─ [用户点击"继续补充"] → 进入文字输入页追加
  │     │     └─ [用户点击"生成文章"] → 进入配置页
  │     │
  │     └─ [识别失败]
  │           展示错误提示
  │           ├─ "重试" → 重新识别
  │           └─ "键盘输入" → 进入文字输入页
  │
  ├─ [用户选择"快速生成"（跳过配置）]
  │     └─ 直接进入生成中页面
  │
  └─ [用户点击"配置 & 生成"]
        └─ 进入配置 Sheet
              │
              ├─ 选择文章类型/语气/长度/读者
              └─ [点击"开始生成"]
                    └─ 进入生成中页面
```

### 2.3 流程二：文字输入生成

```
首页
  │
  └─ [用户点击"键盘输入" 或 向上滑动]
        │
        ▼
  文字输入页
        │
        ├─ 输入标题（可选）
        ├─ 输入内容（必填）
        │     Markdown 快捷工具栏辅助
        │     实时字数统计
        │
        ├─ [导航栏 右上角 "快速配置"]
        │     弹出紧凑菜单：文章类型 / 语气 / 长度
        │     可跳过（使用默认值）
        │
        ├─ [点击"生成文章"]
        │     │
        │     ├─ 内容为空 → 按钮禁用，提示"请先输入内容"
        │     └─ 内容不为空 → 进入生成中页面
        │
        └─ [自动保存]
              离开页面时自动保存草稿
              下次进入可恢复
```

### 2.4 流程三：AI 生成过程

```
生成中页面（全屏）
  │
  ├─ 阶段 1：连接 AI
  │     动画：光点开始汇聚
  │     文案："正在理解你的想法…"
  │     │
  │     ├─ 网络不可用 → 错误提示
  │     └─ 网络正常 → 继续
  │
  ├─ 阶段 2：构建大纲（API 处理中）
  │     动画：光点形成轮廓
  │     文案："正在构建文章大纲…"
  │     流式文字开始逐行浮现
  │
  ├─ 阶段 3：撰写正文
  │     动画：文字逐段填满
  │     文案："正在撰写正文内容…"
  │     流式文字持续更新
  │
  ├─ 阶段 4：润色打磨
  │     动画：金色微光在文字上扫过
  │     文案："正在润色打磨表达…"
  │
  ├─ [生成成功]
  │     │
  │     └─ 自动跳转到文章预览页
  │           触觉反馈：成功
  │
  ├─ [生成失败]
  │     │
  │     ├─ API 4xx 错误 → 提示"API Key 无效"
  │     │     └─ [去设置] → 跳转到设置页
  │     │
  │     ├─ 网络超时 → 提示"网络超时"
  │     │     └─ [重试] → 重新发起请求
  │     │
  │     └─ 其他错误 → 提示"生成失败"
  │           └─ [重试] / [返回]
  │
  └─ [用户取消（3 秒后可见）]
        │
        └─ 二次确认："确定取消生成？"
              ├─ 取消 → 返回生成
              └─ 确定 → 返回首页（保留输入内容）
```

## 4. 边缘场景处理

### 4.1 应用中断

| 中断类型 | 场景 | 处理 |
|----------|------|------|
| 来电 | 正在录音 | 系统自动暂停，通话结束后可恢复 |
| 来电 | 正在 AI 生成 | 后台继续（短时间），超时则提示 |
| 切换 App | 任何页面 | 自动保存当前草稿/状态 |
| 锁屏 | 正在录音 | iOS 限制，录音自动停止 |
| 低电量 | AI 生成中 | 不中断，但提示用户 |
| 内存压力 | 后台过久 | 下次启动恢复到最后状态 |

### 4.2 网络场景

| 场景 | 处理 |
|------|------|
| 无网络 + 录音 | 设备端语音识别可用（离线） |
| 无网络 + AI 生成 | 错误提示，输入内容保留 |
| 弱网络 + AI 生成 | 延长超时到 30s，显示"网络较慢" |
| Wi-Fi → 蜂窝切换 | URLSession 自动处理（waitsForConnectivity） |

### 4.3 数据场景

| 场景 | 处理 |
|------|------|
| 存储空间不足 | 保存前检查，提示清理空间 |
| iCloud 未登录 | iCloud 同步静默关闭，本地可用 |
| iCloud 冲突 | SwiftData 自动合并（last-write-wins） |
| 大量历史文章（> 500） | 分页加载，列表性能优化 |
| 文章被意外删除 | 无回收站，确认删除前明确提示 |

## 5. 关键交互细节

### 5.1 手势定义

| 手势 | 位置 | 行为 |
|------|------|------|
| 按住 | 录音按钮 | 开始录音 |
| 松开 | 录音按钮 | 停止录音 |
| 点击 | 录音按钮 | 切换录音/停止（点击模式） |
| 上滑 | 首页 | 打开文字输入页 |
| 下滑 | 文字输入页 | 收起键盘/返回 |
| 左滑 | 历史文章卡片 | 显示操作按钮 |
| 长按 | 历史文章卡片 | 进入多选模式 |
| 点击文章正文 | 预览页 | 切换沉浸模式 |
| 右滑/返回 | 任何子页面 | 返回上一级 |
