# Aidear 产品需求文档

---

## 1. 产品愿景

**让每个人都能轻松表达观点，用 AI 把碎片想法变成好文章。**

Aidear = AI + Idea。用户只需说出或写下零散的想法、观点、关键词，AI 即可理解其意图，自动补全逻辑、润色表达、扩展内容，最终生成一篇结构完整、可读性强的公众号文章。

---

## 2. 目标用户与场景

### 核心用户画像

| 画像 | 特征 | 核心痛点 |
|------|------|----------|
| **灵感型创作者** | 脑子里想法多，但写出来就散了 | 想法→文字转化困难 |
| **效率型运营** | 需要高频产出公众号内容 | 写作效率低，时间不够 |
| **口语表达者** | 能说会道但不擅长写 | 口语强、书面弱 |
| **轻度写作者** | 偶尔想写点东西分享观点 | 缺乏写作训练，不知从何下笔 |

### 典型使用场景

1. **通勤灵感**：在地铁上突然有个观点，打开 Aidear 用语音说 30 秒，到公司时一篇文章已经生成好。
2. **会议复盘**：开完会有一堆想法，语音录一段，AI 整理成干货文章。
3. **深夜思考**：睡前有些感悟，键盘敲几个关键词和短句，AI 扩展成完整文章。
4. **日更压力**：公众号需要日更，用 Aidear 快速把当天想法转化成文章。

---

## 3. 产品定位

- **输入形式**：语音 + 文字，以最自然的方式捕获想法
- **核心价值**：AI 理解碎片化输入，输出结构化的优质文章
- **输出标准**：符合微信公众号阅读习惯的排版和文体
- **产品调性**：极简、温暖、智能，像有一个编辑朋友在帮你

## 4. 功能需求

### 4.1 想法录入（P0）

#### 4.1.1 语音录入
- 一键开始录音，松手结束或点击停止
- 实时语音波形动画反馈
- 录音结束后自动转文字，支持编辑修正识别结果
- 支持分段录音（继续说…）
- 最长录音时长：5 分钟（超过提示）
- 权限处理：首次使用请求麦克风权限，含说明引导

#### 4.1.2 文字录入
- 简洁文本编辑器，支持 Markdown 基础语法
- 输入提示文案："写下你的想法、观点、关键词…"
- 支持从剪贴板粘贴
- 显示字数统计
- 自动保存草稿（离开页面不丢失）

#### 4.1.3 混合输入
- 语音转文字后可继续键盘编辑补充
- 文字输入时可随时切换到语音追加

### 4.2 AI 文章生成（P0）

#### 4.2.1 生成前配置
- **文章类型**：观点文 / 干货文 / 故事文 / 资讯文 / 自由风格
- **语气风格**：正式专业 / 轻松亲切 / 犀利有料 / 温暖治愈
- **目标长度**：短文（~800字）/ 中篇（~1500字）/ 长文（~2500字）
- **读者对象**：普通读者 / 行业人士 / 年轻群体
- 提供"快速生成"模式：跳过配置，AI 自动判断最佳风格

#### 4.2.2 生成过程
- 分阶段显示生成进度：
  1. 理解想法…
  2. 构建大纲…
  3. 撰写正文…
  4. 润色打磨…
- 优雅的加载动画（灵感流动、文字浮现）

#### 4.2.3 生成结果
- 输出完整文章：**标题 + 引语 + 正文 + 结尾**
- 标题提供 3 个备选
- 支持一键重新生成
- 支持对特定段落重新生成

### 4.3 文章编辑（P0）

- 全文预览，所见即所得
- 支持直接编辑修改任意文字
- Markdown 快捷工具栏（加粗、标题、引用、分割线）
- 撤销/重做
- 字数统计
- 自动保存

### 4.4 对话式调整（P1）

- 以聊天形式对文章进行调整
- 示例指令：
  - "把开头改得更吸引人"
  - "第三段太啰嗦了，精简一下"
  - "加一个名人名言作为结尾"
  - "整体语气再幽默一点"
- AI 理解上下文，局部精准修改
- 支持多轮对话，每次修改保留历史版本

### 4.5 导出与分享（P0）

- 一键复制全文（含排版格式）
- 直接跳转分享到微信（文本 + 格式）
- 导出为图片（适合发朋友圈预览）
- 保存到系统备忘录
- 导出为 Markdown 文件

### 4.6 历史记录（P0）

- 按时间倒序展示所有文章
- 每篇文章显示：标题、摘要、生成时间、风格标签
- 支持搜索（按标题/正文关键词）
- 左滑删除，长按多选批量删除
- 点击进入文章详情，可继续编辑或重新导出
- iCloud 同步（P1）

### 4.7 草稿管理（P1）

- 自动保存未完成的输入和生成结果
- 草稿列表独立展示
- 手动保存/删除草稿

### 4.9 直接 Markdown 转换（P1）

- 用户已有 Markdown 格式文章文本，可直接粘贴转换，无需调用 AI 生成
- 输入原始 Markdown 文本，点击转换后实时渲染为微信排版预览
- 支持一键复制微信格式 HTML，粘贴到公众号编辑器
- 与 AI 生成模式共用同一套 md2wechat 渲染引擎
- 自动提取 Markdown 首行 `# 标题` 作为文章标题展示
- 适用场景：已有 Markdown 稿源、从其他编辑器导出、批量处理已有文章

### 4.8 设置（P0）

- AI 服务配置（API Key 输入、服务商选择）
- 默认文章风格偏好
- 语音识别语言设置
- 外观（跟随系统 / 浅色 / 深色）
- 触觉反馈开关
- 隐私政策与用户协议
- 关于 Aidear

## 5. 非功能需求

### 5.1 性能
- 应用启动时间 ≤ 1.5 秒
- 语音转文字延迟 ≤ 500ms
- AI 生成文章时间 ≤ 15 秒（取决于 API）
- 页面切换帧率 ≥ 60fps

### 5.2 安全
- API Key 存储在 Keychain，不落盘明文
- 用户文章数据仅存储在本地和用户自己的 iCloud
- 不上传用户内容到任何第三方服务器（除用户配置的 AI 服务商外）

### 5.3 可用性
- 支持 VoiceOver 无障碍
- 支持 Dynamic Type（最小到辅助功能大号字体）
- 颜色对比度满足 WCAG AA 标准
- 核心功能在 3 次点击内可达

### 5.4 兼容性
- 最低支持 iOS 17.0
- 适配 iPhone（竖屏为主，横屏支持）
- iPad 适配（P2）
- 深色模式完整支持

## 7. 不做的事（Out of Scope）

- ❌ 不做 AI 生图（公众号封面图等 → P2 考虑）
- ❌ 不做社区/社交功能
- ❌ 不做付费订阅（V1.0 免费 + 自带 API Key）
- ❌ 不做网页版/Android 版（先专注 iOS）
- ❌ 不做文章定时发布（微信接口不开放）
- ❌ 不做多语言文章（先做中文）

## 8. 风险与假设

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| AI API 不稳定 | 核心功能不可用 | 支持多服务商（OpenAI / Anthropic），失败重试机制 |
| 语音识别准确率低 | 用户体验差 | 使用设备端 SFSpeechRecognizer，支持手动修正 |
| App Store 审核 | 上架延迟 | 合规处理，隐私说明完善 |
| 用户不愿配置 API Key | 无法使用核心功能 | 考虑 V2 内置 AI 服务（需商业模式支撑） |

## 9. 已实现功能记录

### 9.1 智能布局切换（2026-05-15）

生成结果返回后，文本输入区和生成按钮自动折叠为顶部紧凑横条（单行显示输入文字），文章内容占据页面主要区域。用户可点击横条重新展开编辑区，修改文字后一键"重新生成"。此交互将输入与阅读分离，让生成结果成为视觉焦点。

### 9.2 自定义 AI Prompt（2026-05-27）

在设置页新增 AI Prompt 编辑区，用户可查看和修改默认的 AI system prompt。修改保存后立即生效，所有新生成的文章使用自定义 prompt。支持「恢复默认」按钮一键还原。

### 9.3 直接 Markdown 转换模式（2026-05-16）

新增"直接转换"模式，与"AI 创作"模式并列。用户在模式选择器中切换到"直接转换"，输入已写好的 Markdown 文本，点击转换后直接渲染为微信排版预览，不经过 AI 调用。此模式适用于已有 Markdown 稿源、从其他编辑器导出内容、或批量处理已有文章的场景。渲染引擎与 AI 生成模式完全相同（md2wechat CSS）。
