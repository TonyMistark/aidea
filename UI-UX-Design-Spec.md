# Aidear UI/UX 设计规范

---

## 1. 设计哲学

### 核心理念

**"想法如流水，文章如雕刻"**

Aidear 的设计遵循三个核心原则：

1. **极致的输入体验** — 消除一切摩擦，让用户以最自然的方式倾泻想法
2. **温暖的 AI 陪伴感** — AI 不只是工具，更像一个懂你的编辑朋友
3. **iOS 原生之美** — 深度融入 Apple 设计语言，让用户感觉这是 iOS 本该有的应用

### 情感关键词

温暖 · 专注 · 流畅 · 信赖 · 聪明

| `#F59E0B` | Primary | 主按钮、选中态、品牌色 |

| `#FCD34D` | Primary Light | 浅色背景、标签 |
| `#FEF3C7` | Primary Ultra Light | 大块浅色背景 |

代表 AI 的智能与可靠

| 色值 | 名称 | 用途 |

### 2.4 语义色

| 用途 | 色值 |
|------|------|
| 成功 | `#34C759` |
| 警告 | `#FF9500` |
| 错误 | `#FF3B30` |
| 信息 | `#007AFF` |

### 2.5 渐变

**主渐变（用于大按钮、关键页面）**：
- 方向：135°（左上到右下）
- 起点：`#F59E0B`（琥珀金）
- 终点：`#F97316`（暖橙）
- 用途：AI 生成按钮、品牌装饰

**AI 生成中渐变（用于加载动画）**：
- 方向：90°（左到右）流动
- 起点：`#FCD34D` → 中点：`#F59E0B` → 终点：`#0EA5E9`
- 用途：表示 AI 正在思考/生成

## 4. 空间系统

### 4.1 间距（8pt 网格）

| Token | 值 | 用途 |
|-------|-----|------|
| `spacing-xxs` | 4pt | 紧密关联元素（图标-文字） |
| `spacing-xs` | 8pt | 标签组间距、列表项内部 |
| `spacing-sm` | 12pt | 卡片内边距（小） |
| `spacing-md` | 16pt | 标准边距、列表项间距 |
| `spacing-lg` | 20pt | 卡片内边距（大） |
| `spacing-xl` | 24pt | 段落间距 |
| `spacing-xxl` | 32pt | 区块间距 |
| `spacing-xxxl` | 40pt | 大区块分隔 |
| `spacing-huge` | 48pt | 页面顶部/底部留白 |

### 4.2 安全区域

- 内容区域左右边距：`spacing-md`（16pt）或 `spacing-lg`（20pt）
- 顶部：尊重 safeAreaInsets，导航栏下方 16pt
- 底部：尊重 safeAreaInsets，TabBar 上方 16pt
- 卡片/列表：左右各 20pt（在 iPhone 上形成自然内边距）

### 4.3 圆角

| Token | 值 | 用途 |
|-------|-----|------|
| `radius-sm` | 8pt | 小按钮、标签 Chip、输入框 |
| `radius-md` | 12pt | 卡片、列表行 |
| `radius-lg` | 16pt | 模态框、Sheet、大卡片 |
| `radius-xl` | 20pt | 底部 Sheet |
| `radius-full` | 9999pt | 胶囊按钮、头像 |

## 6. 动效系统

### 6.1 过渡动画

| 场景 | 动效 | 时长 | 曲线 |
|------|------|------|------|
| 页面推入 | NavigationStack push | 系统默认 | 系统默认 |
| 页面弹出 | NavigationStack pop | 系统默认 | 系统默认 |
| Modal 弹出 | `.sheet` 或 `.fullScreenCover` | 系统默认 | spring |
| 按钮按下 | 缩放 0.97 → 1.0 | 0.15s | easeOut |
| 列表项出现 | 从下淡入 + 位移 | 0.3s | easeOut（交错 0.05s） |
| 卡片出现 | 缩放 0.95→1.0 + 淡入 | 0.4s | spring(damping: 0.7) |

### 6.2 微交互

| 场景 | 动效 |
|------|------|
| **录音按钮** | 按下时外圈扩散脉冲（5 层涟漪，间隔 0.8s），中间麦克风图标微微放大 |
| **AI 生成中** | 流动的金→蓝渐变光条从左到右扫过，文字逐行浮现 |
| **生成完成** | 轻微弹性动画（spring），标题高亮闪一下 |
| **复制成功** | SF Symbol `checkmark` 弹入 + 自动消失（1.5s） |
| **删除** | 卡片缩小 + 向左滑出 + 淡出 |
| **切换模式** | 录音→键盘：淡入淡出交叉溶解（0.25s） |

### 6.3 触觉反馈

| 场景 | 反馈类型 | 强度 |
|------|----------|------|
| 开始录音 | `notification(.warning)` | 重 |
| 停止录音 | `notification(.success)` | 中 |
| 生成完成 | `notification(.success)` | 中 |
| 按钮点击 | `impact(.light)` | 轻 |
| 删除确认 | `impact(.heavy)` | 重 |
| 滑动操作 | `selection` | 轻 |
| 复制成功 | `impact(.medium)` | 中 |
| 生成失败 | `notification(.error)` | 重 |

## 7. 页面详细设计

### 7.1 启动屏（Launch Screen）

- App Logo 居中：琥珀金色 "Aidear" 字标 + 微光粒子
- 背景色：系统背景色
- 无多余元素
- 显示时长：≤ 1 秒

---

### 7.2 引导页（Onboarding）

**只在首次启动时显示，共 3 页，可滑动，底部 Skip 按钮。**

#### 第 1 页：说出想法
- 插图：声波动画 + 漂浮的文字泡泡
- 标题：**"说出你的想法"**
- 副标题：用语音或键盘记录碎片想法，不需要完美的句子
- 色调：琥珀金为主

#### 第 2 页：AI 妙笔
- 插图：金色光点流入文字排版中
- 标题：**"AI 帮你润色"**
- 副标题：AI 理解你的意图，整理逻辑，扩展内容，打磨表达
- 色调：金蓝渐变

#### 第 3 页：生成文章
- 插图：完整排版好的文章从屏幕中浮现
- 标题：**"一键生成文章"**
- 副标题：获得一篇可直接发布的公众号文章，随时分享
- 色调：知性蓝为主
- 底部按钮：**"开始使用"**（主色大按钮）

**UI 细节**：
- 每页切换时插图元素有微动效
- 页面指示器使用金色圆点
- "跳过"按钮在右上角，灰色文字，不抢眼
- "开始使用"按钮有微光动画吸引点击

#### 录音按钮设计（核心交互元素）

- **外观**：圆形按钮，直径 80pt
- **默认态**：白色填充 + 琥珀金描边（2pt）+ `mic.fill` 图标 + 底部文字"按住说话"

- **录音中（松开后持续录音）**：显示录音时长，按钮变为红色停止按钮，波形实时响应
- **禁用态**（无麦克风权限）：灰色，点击弹出权限引导

- 录音结束后出现，显示识别文字
- 右上角编辑图标，可点击修正
- 底部两个按钮：
  - "继续补充"（outline 样式）
  - "生成文章"（主色填充，带 → 箭头）

#### 文字输入替代

- 点击"键盘输入"或向上滑，进入全屏文字编辑
- 或作为底部 Sheet 弹出（iOS 风格）

#### 空态（首次使用）

- 录音按钮保持居中
- 上方无波形动画，改为引导文字："试试按住按钮，说出你今天的第一个想法"
- 文字轻柔淡入

### 7.6 AI 生成中页（Generating）

#### 布局

全屏沉浸式体验，不可关闭（但可以后台）。

```
┌─────────────────────────────────┐
│                                 │
│                                 │
│          [AI 动画区域]           │
│                                 │
│      ◇ ← 金色光点汇聚 → ◇       │
│         ✦ 粒子浮动 ✦            │
│                                 │
│     正在理解你的想法...          │  ← 动态切换的文案
│                                 │
│  ┌───────────────────────────┐  │
│  │ · · · · · · · · · · · · ·│  │  ← 文字浮现区域
│  │ ·   浮现中的文字片段...   │  │
│  │ · · · · · · · · · · · · ·│  │
│  └───────────────────────────┘  │
│                                 │
│      [进度指示器 ••••····]      │
│                                 │
│       取消（小字，30s 后出现）    │
│                                 │
└─────────────────────────────────┘
```

#### 动画设计

**核心动画：灵感汇聚**

1. 多个小金色光点从屏幕四周向中心汇聚
2. 汇聚后形成旋转的文字轮廓
3. 文字轮廓逐渐填充，碎片式文字逐行浮现
4. 浮现的文字是真实的正在生成的内容片段（流式展示）

**阶段文案（循环切换 + 进度）：**
- "正在理解你的想法…"
- "正在构建文章大纲…"
- "正在撰写正文内容…"
- "正在润色打磨表达…"

**进度指示器**：用 4 个金色圆点，依次亮起再全部闪烁表示工作

#### 取消操作

- 前 3 秒内：没有取消按钮（防止误触）
- 3 秒后：底部出现小字"取消生成"，点击后二次确认

## 11. 已实现 UI 变更记录

### 11.1 动态布局：输入区折叠（2026-05-15）

**交互流程**：

1. **生成前**：文本编辑器（最小高度 150pt）+ "AI 生成文章"按钮位于页面中央
2. **生成中**：按钮显示 `ProgressView` + "AI 正在创作..."，按钮禁用
3. **生成完成**：
   - 输入区自动折叠（`easeInOut 0.25s` 动画）
   - 顶部出现紧凑横条 — 左侧铅笔图标 + 单行输入文字预览 + 右侧展开/收起箭头
   - 生成按钮文字变为"重新生成"
   - 文章内容占据页面主区域，WebView 高度自适应内容
4. **点击横条**：展开/收起输入编辑区（带动画），用户可修改文字后重新生成
5. **重新生成**：生成新结果后输入区再次自动折叠

**设计意图**：将"输入"与"阅读"两个阶段分离。创作时输入区为主，阅读时文章为主。折叠态保留了编辑入口，但不抢占阅读注意力。
