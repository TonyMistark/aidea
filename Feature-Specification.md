## 2. 功能详细规格

### F01：语音录音与转文字

**描述**：用户通过按住或点击录音按钮，录制语音，系统自动转换为文字。

**交互流程**：
1. 用户按住圆形录音按钮（或点击一次开始，再点击停止）
2. 录音按钮呈现按下态（放大 + 涟漪动画 + 填充主色）
3. 实时显示录音时长和声波动画
4. 松开按钮（或点击停止），录音结束
5. 自动调用 SFSpeechRecognizer 进行语音识别
6. 识别结果展示在输入区域，等待用户确认或编辑
7. 用户可以继续追加录音

**状态处理**：
| 状态 | 展示 |
|------|------|
| 未授权麦克风 | 录音按钮灰色，点击弹出权限引导 |
| 录音中 | 按钮变红/填充主色，声波动画活跃 |
| 识别中 | "识别中…"加载指示器 |
| 识别成功 | 文字淡入展示，编辑按钮出现 |
| 识别失败 | "语音识别失败"，提供键盘输入入口 |
| 网络离线 | 使用设备端离线识别（iOS 内置能力） |

**边界条件**：
- 最长录音 5 分钟，达到 4:50 时震动提示
- 录音过程中来电/闹钟 → 系统中断，自动保存已录音部分
- 识别结果为空（无有效语音） → 提示"没有识别到语音内容"

### F02：键盘文字输入

**描述**：用户通过键盘输入想法、观点、关键词。

**交互流程**：
1. 从首页点击"键盘输入"或语音识别后点击编辑
2. 进入全屏文本编辑页面
3. 输入内容，实时显示字数
4. 可随时点击"生成文章"进入配置页

**状态处理**：
| 状态 | 展示 |
|------|------|
| 无内容 | Placeholder："写下你的想法、观点、关键词…" |
| 有内容 | 正常文本展示，底部显示字数 |
| 达到目标字数 | 字数处显示 ✓ 标记 |

**边界条件**：
- 输入内容上限：10,000 字
- 超过 5,000 字时提示"内容较长，AI 可能需要更长时间处理"
- 输入内容为空时 "生成文章" 按钮禁用

### F04：AI 文章生成

**描述**：将用户输入送交 AI 服务，生成结构化公众号文章。

**输入**：
- 原始文本（语音识别结果 / 键盘输入）
- 配置参数（文章类型、语气、长度、读者对象）
- 对话历史上下文（如果是微调模式）

**输出**：
- 文章标题（1 个主标题 + 2 个备用标题）
- 文章引语/摘要
- 文章正文（含小标题、段落、引用等）
- 结尾段落

**处理流程**：
1. 构建 System Prompt（含用户配置）
2. 发送 API 请求（支持流式响应 SSE）
3. 流式解析内容并逐步展示
4. 完成后解析出标题、引语、正文
5. 保存到本地数据库

**错误处理**：
| 错误 | 处理 |
|------|------|
| 网络超时（15s） | 提示"网络超时"，可重试 |
| API 返回 4xx | 提示"API Key 无效或额度不足，请检查设置" |
| API 返回 5xx | 提示"AI 服务暂时不可用"，可重试 |
| 流式中断 | 保留已生成部分，提示可继续或重新生成 |
| 响应内容为空 | 提示"未能生成文章"，建议调整输入后重试 |

### F05：生成前风格配置

**描述**：在 AI 生成前，用户可以选择文章类型、语气、长度等参数。

**文章类型**：
| 类型 | 描述 | AI Prompt 指引 |
|------|------|----------------|
| 观点文 | 表达个人观点和见解 | 强调逻辑推理、论据支撑、结论明确 |
| 干货文 | 提供实用信息和知识 | 强调条理清晰、知识点突出、可操作性 |
| 故事文 | 以故事形式叙述 | 强调叙事性、场景感、情感共鸣 |
| 资讯文 | 传递信息和动态 | 强调客观、简洁、信息密度 |
| 自由风格 | AI 自行判断 | 综合判断，灵活发挥 |

**语气风格**：
| 风格 | 描述 |
|------|------|
| 正式专业 | 严谨、客观、有深度 |
| 轻松亲切 | 像朋友聊天，有温度 |
| 犀利有料 | 一针见血，观点鲜明 |
| 温暖治愈 | 柔和、感性、有共鸣 |

**目标长度**：
| 长度 | 字数范围 | 适用场景 |
|------|----------|----------|
| 短文 | 600-1000 字 | 快速阅读、碎片时间 |
| 中篇 | 1200-1800 字 | 标准公众号文章 |
| 长文 | 2000-3000 字 | 深度内容 |

**读者对象**：
- 普通读者（通俗易懂）
- 行业人士（可用专业术语）
- 年轻群体（网络化表达）

**默认值（用户不选时）**：
- 类型：自由风格
- 语气：轻松亲切
- 长度：中篇（~1500 字）
- 读者：普通读者

### F07：文章预览

**描述**：以精美的排版展示 AI 生成的文章，支持阅读模式滚动浏览。

**内容区块（从上到下）**：
1. 文章标题（24pt Bold，居中或左对齐）
2. 引语/摘要（17pt Italic，Secondary Label 颜色）
3. 分隔线
4. 正文内容（含 Markdown 渲染：H2、段落、列表、引用、加粗、链接）
5. 结尾

**Markdown 渲染规则**：
| Markdown | 渲染效果 |
|----------|----------|
| `## 标题` | 19pt Semibold，上下各 16pt 间距 |
| `**加粗**` | 17pt Semibold |
| `> 引用` | 左侧 3pt 主色竖线 + 缩进 + 16pt 灰色文字 |
| `- 列表` | 圆点缩进列表 |
| `1. 有序` | 数字缩进列表 |
| `---` | 居中分割线 |
| `` `代码` `` | 等宽字体，浅灰背景 |

**交互**：
- 点击文章任意位置 → 显示/隐藏导航栏和工具栏（沉浸式阅读）
- 长按文字 → 系统菜单（选择/复制/查询）
- 下拉顶部 → 关闭预览（返回）

### F10/F11/F12：导出与分享

**一键复制**：
- **摘要复制**：摘要文字右侧有独立复制按钮，一键复制摘要纯文本
- **正文复制**：复制带微信排版样式的 HTML，粘贴到公众号编辑器保留样式
- 复制后 Toast："已复制到剪贴板" + checkmark 图标
- 考虑提供"复制为 Markdown"和"复制为纯文本"两个选项

**分享到微信**：
- 检测是否安装微信
  - 已安装：通过 URL Scheme 跳转微信，传递文本
  - 未安装：提示"未安装微信"，回退到系统分享

**系统分享 Sheet**：
- 使用 `UIActivityViewController`
- 排除不需要的分享目标
- 支持 AirDrop、备忘录、邮件等

**导出为图片（P1）**：
- 将文章渲染为长图
- 适配手机屏幕宽度（375pt 基准）
- 顶部包含 Aidear 品牌水印（可选关闭）
- 保存到相册或分享

### F13/F14/F15：历史记录

**列表排序**：按"最后修改时间"倒序

**列表项信息**：
- 标题（主文字，1 行截断）
- 摘要（次要文字，2 行截断）
- 生成时间（相对时间：刚刚/5 分钟前/2 小时前/昨天/3 天前/日期）
- 风格标签（圆角 Chip）
- 字数

**筛选功能**：
- 按文章类型筛选（横向滑动 Chips，可多选）
- 筛选后列表实时更新

**搜索功能**：
- 搜索范围：标题 + 正文（全文搜索）
- 实时搜索结果（输入过程中即开始筛选）
- 关键词高亮（搜索结果中）

**删除操作**：
- 单条删除：左滑 → 确认删除
- 批量删除：长按进入编辑模式 → 勾选 → "删除选中"
- 删除后不可恢复（不做回收站，简化设计）

### F33：直接 Markdown 转换

**描述**：用户已有 Markdown 格式文章文本，可直接粘贴并转换为微信排版预览，不调用 AI 生成。

**适用场景**：
- 从其他 Markdown 编辑器导出的文章
- 已有稿件需要快速查看微信排版效果
- 批量处理已有文章（无需 AI 创作）

**交互流程**：
1. 用户在模式选择器切换到"直接转换"
2. 将 Markdown 文本粘贴到输入区
3. 点击"转换"按钮
4. 文章内容即时渲染为微信排版预览
5. 自动提取 Markdown 首行 `# 标题` 作为文章标题展示
6. 用户可点击"复制微信格式"将带样式 HTML 复制到剪贴板

**模式选择**：
- 顶部 Segmented Control：`AI 创作 | 直接转换`
- 切换模式时输入内容保留（不清空）
- 两种模式共用同一套 md2wechat CSS 渲染引擎

**与 AI 生成模式的区别**：
| | AI 生成 | 直接转换 |
|---|---|---|
| 输入 | 碎片想法/关键词 | 完整 Markdown 文本 |
| 处理 | 调用 AI API | 客户端即时渲染 |
| 网络依赖 | 需要 | 不需要 |
| 输出元数据 | 标题 + 摘要 + 封面图提示词 | 仅提取 `# 标题` |
| 耗时 | 数秒到数十秒 | 即时 |

**状态处理**：
| 状态 | 展示 |
|------|------|
| 输入为空 | "转换"按钮灰色禁用 |
| 有内容 | "转换"按钮蓝色可点击 |
| 转换完成 | 输入区折叠，文章预览展示 |

**边界条件**：
- 输入无 `# 标题` → 文章无标题展示，仅渲染正文
- Markdown 语法错误 → 浏览器尽力渲染（marked.js 容错）
- 超大内容（>50KB）→ WebView 渲染延迟可接受，无额外处理

### F34：多 Prompt 管理

**描述**：在设置页管理多个 AI Prompt（添加/编辑/复制/删除），通过点击切换当前生效的 Prompt，灵活定制文章生成风格。

**交互流程**：
1. 用户进入设置页，找到「AI Prompt」区域
2. 列表中展示所有已创建的 Prompt，每行显示名称 + 内容预览
3. **点击整行** → 设为当前生效的 Prompt（左侧圆圈变为蓝色 checkmark）
4. **左滑某行** → 出现三个操作按钮：
   - **复制**：复制该 Prompt 为一条新记录，名称带"副本"标记
   - **编辑**：打开编辑 sheet，可修改名称和内容，含"恢复默认"按钮
   - **删除**：弹出二次确认，删除后不可撤销（仅剩 1 条时禁止删除）
5. 点击「添加 Prompt」→ 打开新建 sheet，填写名称和内容后保存，自动设为当前生效
6. 切换/修改后对所有后续 AI 生成立即生效

**边界条件**：
- Prompt 名称不能为空，否则不允许保存
- 只剩最后一条 Prompt 时禁止删除
- 删除当前生效的 Prompt 后，自动切换到列表中第一条
- 副本 Prompt 名称带"（副本）"标记，用户可随时修改

### F35：AI 生成取消

**描述**：AI 生成过程中，用户可点击按钮取消当前生成任务，避免浪费 token。

**交互流程**：
1. AI 生成过程中，按钮变为「取消」状态
2. 用户点击取消按钮，弹出二次确认对话框
3. 确认后中断 API 请求，显示「已取消生成」提示
4. 已生成的部分内容保留在结果区，用户可选择重新生成

**状态处理**：
| 状态 | 展示 |
|------|------|
| 生成中 | 按钮显示「AI创作中(Xs)」，点击可取消 |
| 取消确认 | 二次确认弹窗 |
| 已取消 | 提示「已取消生成」，按钮恢复为「重新生成」 |

---

### F36：生成计时器

**描述**：AI 生成过程中，按钮上实时显示已用秒数，让用户直观感知生成耗时。

**交互流程**：
1. 点击「开始AI创作」后，按钮文字变为「AI创作中(1s)」
2. 每秒更新计时数字
3. 生成完成或取消后计时停止，按钮恢复常态
