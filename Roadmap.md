## 2. V1.0 — MVP（8-10 周）

### 目标
闭环核心体验：**语音/文字输入 → AI 生成 → 编辑 → 分享**

### 功能范围

| 模块 | 功能 | 优先级 |
|------|------|--------|
| 🎤 想法录入 | 语音录音 + 转文字 | P0 |
| ⌨️ 想法录入 | 键盘文字输入 | P0 |
| 🔀 想法录入 | 语音/文字混合输入 | P0 |
| 🤖 AI 引擎 | OpenAI + Anthropic 双服务商 | P0 |
| ⚙️ AI 引擎 | 风格/语气/长度配置 | P0 |
| 🎬 AI 引擎 | 流式生成 + 生成动画 | P0 |
| 📄 文章 | 预览（Markdown 渲染） | P0 |
| ✏️ 文章 | 手动编辑 | P0 |
| 📤 导出 | 复制全文 | P0 |
| 📤 导出 | 系统分享 Sheet | P0 |
| 📤 导出 | 跳转微信分享 | P0 |
| 📋 历史 | 文章列表 + 搜索 + 删除 | P0 |
| ⚙️ 设置 | AI 服务配置 + API Key | P0 |
| ⚙️ 设置 | 默认偏好设置 | P0 |
| 🎨 系统 | 浅色/深色模式 | P0 |
| 🚀 系统 | 引导页（Onboarding） | P0 |
| 🔒 系统 | 权限引导 + Keychain 安全 | P0 |

### 里程碑

| 里程碑 | 时间 | 交付物 | 标准 |
|--------|------|--------|------|
| M1 — 基础架构 | Week 1-2 | 项目搭建 + 数据模型 + 导航框架 | Tab Bar 可切换，数据可持久化 |
| M2 — 输入模块 | Week 3-4 | 语音录音 + 转文字 + 键盘输入 | 语音→文字可闭环 |
| M3 — AI 引擎 | Week 5-6 | AI 服务集成 + 流式生成 + 配置页 | 输入→生成→预览可闭环 |
| M4 — 文章编辑 | Week 7 | 文章预览 + 编辑 + 备用标题 | 生成后可编辑保存 |
| M5 — 导出分享 | Week 7 | 复制 + 分享 + 微信跳转 | 文章可导出到微信 |
| M6 — 辅助模块 | Week 8 | 历史记录 + 搜索 + 设置 + 引导页 | 全部 P0 功能可用 |
| M7 — 打磨验证 | Week 9-10 | 测试 + Bug 修复 + 性能优化 + TestFlight | 可提交 App Store 审核 |

### 成功标准

- [ ] 从"打开 App"到"复制文章到微信"全流程 ≤ 60 秒
- [ ] P0 功能无阻塞 Bug
- [ ] TestFlight 内测用户 NPS ≥ 30
- [ ] App Store 审核一次通过

| 里程碑 | 时间 | 关键交付 |

## 7. 风险与依赖

| 风险 | 影响版本 | 缓解 |
|------|----------|------|
| App Store 审核不通过 | V1.0 | 严格遵循 HIG 合规文档，提前准备审核材料 |
| AI API 服务不稳定 | 全部 | 多服务商支持，失败重试 |
| 语音识别中文准确率 | V1.0 | 使用设备端识别 + 手动编辑补救 |
| API Key 配置门槛劝退用户 | V1.0+ | V2.0 考虑内置 AI 服务 |
| SwiftData iCloud 同步 Bug | V1.1 | 充分测试，必要时回退到 Core Data + CloudKit |

---

## 8. 迭代原则

1. **先闭环再完美**：每个版本确保核心流程闭环可用，再追求细节
2. **数据驱动**：通过 TestFlight 用户反馈和使用数据指导优先级
3. **保持简洁**：拒绝功能膨胀，每个新增功能必须通过"是否 80% 用户需要"的检验
4. **原生优先**：能用系统 API 就不用第三方库，降低维护成本
